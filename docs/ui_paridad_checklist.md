# Checklist de Paridad Visual 1:1

Antes de marcar cualquier fase como "Done", asegúrate de que todos los componentes y rutas cumplen con las especificaciones visuales del prototipo.

## Shell (global)
- [ ] Sidebar: Asegurarse que el ancho, padding, tipografía e iconografía coinciden exactamente con el prototipo.
- [ ] Header: Comprobar que la alineación y los elementos coinciden exactamente (título "NeuroCampus").
- [ ] Layout: Asegurarse que el grid (12 columnas), márgenes, gaps y alturas son los mismos.

## Componentes
- [ ] Cards: Revisa bordes, sombras y tamaños.
- [ ] Botones: Verificar tamaños, variantes (primario/secondary) y estados (hover, disabled).
- [ ] Tablas: Confirmar el orden de las columnas, la paginación y el comportamiento de filtro.

## Estados
- [ ] Loading: Verificar que los skeletons y placeholders se usan correctamente.
- [ ] Empty: Confirmar que los mensajes y layouts de vacío están alineados con el prototipo.
- [ ] Error: Revisar banners o cards con mensajes de error.
